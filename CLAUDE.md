## Project

Memorial photo slideshow hosted on GitHub Pages. Photos stored on Google Drive, served via `lh3.googleusercontent.com/d/FILE_ID` direct URLs.

## Build

```bash
./build-photos-js.sh FOLDER_ID   # regenerates photos.js from Google Drive folder
```

Drive folder ID: `19lQWvKuKKoZnLJxK1JjjOS2YZlAchBXd`

No other build tools or package managers. Requires python3 + internet.

## Architecture

- `index.html` — self-contained slideshow (HTML/CSS/JS). Splash screen triggers Fullscreen API on user tap, then starts slideshow loop.
- `photos.js` — JS file declaring `const PHOTOS = [...]` array of Google Drive direct URLs. Generated by build script — do not hand-edit.
- `build-photos-js.sh` — fetches Google Drive embedded folder view, parses HTML for file IDs/names, deduplicates, filters unsupported formats, writes `photos.js`.
- `qr.png` — static QR code pointing to GitHub Pages URL.

## Key Config

In `index.html`, timing constants at top of `<script>`:
- `CROSSFADE_MS` — fade duration (must match CSS `transition: opacity` value)
- `SLIDE_MS` — per-photo display time

## Local Testing

Open `index.html` in a browser. Internet required (photos load from Google Drive).

- Deploy: commit and push to `main` — GitHub Pages updates automatically

## Notes

- Crossfade CSS and JS must stay in sync manually
- HEIC files filtered out (no browser support)
- Duplicates should be removed in Google Drive before running build script
- iOS Safari: no Fullscreen API support — slideshow works in normal viewport
- Broken images hidden via `onerror` handler (handles deleted/unshared Drive files)
- Slideshow re-shuffles at end of each loop, avoiding back-to-back repeat of last photo
